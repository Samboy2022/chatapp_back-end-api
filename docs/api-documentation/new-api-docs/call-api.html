POST / GET /stream/token

Stream Video Token (Backend & Flutter usage)

The Stream token endpoint issues short-lived credentials required by the Stream Video SDK to join or create WebRTC rooms. Your Laravel backend must mint tokens (it holds the STREAM_API_SECRET). The client requests a token from the backend and then uses that token to connect to Stream for media transport (audio and/or video).

Backend - Endpoints

POST /api/stream/token — Generate a token for the authenticated user. Accepts optional call_id and room_id in the request body.

GET /api/stream/config — Returns the Stream API key and feature flags the client needs to initialize the SDK.

Example Request (backend)

POST http://127.0.0.1:8000/api/stream/token
Authorization: Bearer {USER_API_TOKEN}
Content-Type: application/json

{}

Example Success Response

{
  "success": true,
  "data": {
    "token": "eyJhbGciOi...",
    "api_key": "your_stream_api_key",
    "user_id": 123,
    "expires_at": "2025-09-24T10:30:00.000000Z",
    "call_id": null,
    "room_id": null
  },
  "message": "Stream video token generated successfully"
}

How it works (summary)

  1. Client authenticates with your Laravel backend and calls /api/stream/token.
  2. Backend validates the request, creates (or retrieves) the call record if applicable, and mints a short-lived Stream token using your STREAM_API_SECRET.
  3. Backend returns the token and Stream API key (via /api/stream/config or embedded in the token response).
  4. Client initializes Stream Video SDK with the API key and token, then creates/joins a room for audio/video transport.

Flutter client example

// 1) Request a Stream token from your backend
final resp = await http.post(
  Uri.parse('$baseUrl/api/stream/token'),
  headers: {
    'Authorization': 'Bearer $token',
    'Content-Type': 'application/json',
  },
  body: jsonEncode({});
);

final body = jsonDecode(resp.body);
final streamToken = body['data']['token'];
final apiKey = body['data']['api_key'];

// 2) Initialize StreamVideoClient and join a call/room
final client = StreamVideoClient(apiKey, user: User.regular(userId: currentUserId));
final call = client.makeCall(callType: StreamCallType.defaultType(), id: 'call_123');
await client.connect(token: streamToken);
await call.join();

Audio-only vs Audio+Video

Use the same token for both modes. To do audio-only, the client simply does not publish a video track (or unpublishes it) — Stream handles media transport. For video calls, publish both audio and video tracks.

Security notes

  • Never expose STREAM_API_SECRET in the client. Mint tokens on the server only.
  • Keep tokens short-lived (the backend issues them with an expiry, typically 1–24 hours).
  • Authorize token requests: only authenticated users should be able to request tokens for themselves. Admin-only endpoints (e.g. /api/stream/token/user) can mint tokens for other users.
POST /calls/{callId}/accept

Accept an incoming call

Must be the call receiver. Call status changes from 'ringing' to 'answered'.

POST /calls/{callId}/reject

Reject an incoming call

Must be the call receiver. Call status changes to 'declined'.

POST /calls/{callId}/end

End an active call

Can be called by either participant. Calculates and stores call duration.

GET /calls

Get user's call history with pagination

Query Parameters

per_page: integer (default: 50)
page: integer (default: 1)
type: audio/video (optional)

Response Format

{
  "success": true,
  "data": {
    "current_page": 1,
    "data": [
      {
        "id": 1,
        "caller_id": 2,
        "callee_id": 1,
        "type": "voice",
        "status": "ended",
        "duration": 245,
        "started_at": "2025-09-15T00:36:00.000000Z"
      }
    ]
  }
}
GET /calls/active

Get all active calls for the user

Returns calls with status 'ringing' or 'answered'.

GET /calls/statistics

Get detailed call statistics

Total calls: Count of all calls
Outgoing: Calls initiated by user
Incoming: Calls received by user
Answered: Successfully answered calls
Missed: Unanswered incoming calls
Talk time: Total duration formatted
Video calls: Video call count
Audio calls: Audio call count
GET /calls/missed-count

Get count of missed calls in the last 7 days

Useful for displaying notification badges in mobile apps.

🔄 Call Lifecycle

Ringing

Call initiated, waiting for receiver

Answered

Call accepted and active

Declined

Receiver rejected the call

Ended

Call completed normally

💻 Code Examples

JavaScript (Call Management)

// Initiate voice call
const initiateCall = async (receiverId, callType = 'voice') => {
  const response = await fetch('http://127.0.0.1:8000/api/calls', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      receiver_id: receiverId,
      type: callType
    })
  });
  return await response.json();
};

// Answer incoming call
const answerCall = async (callId) => {
  const response = await fetch(`http://127.0.0.1:8000/api/calls/${callId}/accept`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// End call
const endCall = async (callId) => {
  const response = await fetch(`http://127.0.0.1:8000/api/calls/${callId}/end`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// Get call history
const getCallHistory = async () => {
  const response = await fetch('http://127.0.0.1:8000/api/calls', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// Get call statistics
const getCallStatistics = async () => {
  const response = await fetch('http://127.0.0.1:8000/api/calls/statistics', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

# End call

  -H "Authorization: Bearer TOKEN"

# Get call statistics
