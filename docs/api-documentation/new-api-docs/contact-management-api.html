6 endpoints • Contact sync • Favorites • Blocking

GET /contacts

Get user's contacts list with pagination

Query Parameters

per_page: integer (default: 50)
page: integer (default: 1)
search: string (optional search)

Returns contacts sorted by favorites first, then alphabetically by name.

POST /contacts/sync

Sync device contacts with app users

Request Body

{
  "contacts": [
    {
      "name": "John Doe",
      "phone": "+1234567890"
    },
    {
      "name": "Jane Smith",
      "phone": "+1234567891"
    }
  ]
}

Automatically matches device contacts with registered app users. Only creates contact records for matched users.

GET /contacts/search

Search for users by name or phone number

Query Parameters

query: string (min 3 characters)
per_page: integer (default: 20)

Searches across all registered users, not just existing contacts.

POST /contacts/favorite/{contactId}

Toggle favorite status for a contact

How it works: Each call toggles the favorite status. Returns the updated contact with the new favorite status.
GET /contacts/favorites

Get user's favorite contacts

Returns only contacts marked as favorites, sorted alphabetically by contact name.

POST /contacts/block/{contactId}

Block a contact/user

Blocking effects: Prevents calls, messages, and contact visibility. Creates mutual blocking to prevent the blocked user from contacting you.
POST /contacts/unblock/{contactId}

Unblock a contact/user

Removes blocking restrictions and restores normal contact functionality.

GET /contacts/blocked

Get user's blocked contacts

Returns all contacts that have been blocked by the user.

💻 Code Examples

JavaScript (Contact Management)

// Sync device contacts
const syncContacts = async (deviceContacts) => {
  const response = await fetch('http://127.0.0.1:8000/api/contacts/sync', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      contacts: deviceContacts
    })
  });
  return await response.json();
};

// Search users
const searchUsers = async (query) => {
  const response = await fetch(`http://127.0.0.1:8000/api/contacts/search?query=${query}`, {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// Toggle favorite
const toggleFavorite = async (contactId) => {
  const response = await fetch(`http://127.0.0.1:8000/api/contacts/favorite/${contactId}`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// Block contact
const blockContact = async (contactId) => {
  const response = await fetch(`http://127.0.0.1:8000/api/contacts/block/${contactId}`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// Get blocked contacts
const getBlockedContacts = async () => {
  const response = await fetch('http://127.0.0.1:8000/api/contacts/blocked', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

cURL Commands

# Get contacts
curl -X GET "http://127.0.0.1:8000/api/contacts" \
  -H "Authorization: Bearer TOKEN"

# Sync device contacts
curl -X POST "http://127.0.0.1:8000/api/contacts/sync" \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "contacts": [
      {"name": "John Doe", "phone": "+1234567890"}
    ]
  }'

# Search users
curl -X GET "http://127.0.0.1:8000/api/contacts/search?query=john" \
  -H "Authorization: Bearer TOKEN"

# Toggle favorite
curl -X POST "http://127.0.0.1:8000/api/contacts/favorite/2" \
  -H "Authorization: Bearer TOKEN"

# Block contact
curl -X POST "http://127.0.0.1:8000/api/contacts/block/2" \
  -H "Authorization: Bearer TOKEN"

# Get blocked contacts
curl -X GET "http://127.0.0.1:8000/api/contacts/blocked" \
  -H "Authorization: Bearer TOKEN"

# Get favorite contacts
curl -X GET "http://127.0.0.1:8000/api/contacts/favorites" \
  -H "Authorization: Bearer TOKEN"

🔧 Integration Notes

  • Request device contact permission before syncing
  • Handle contact sync in background to avoid UI blocking
  • Normalize phone numbers before sync for better matching
  • Cache contact lists locally for better performance
  • Respect privacy settings when displaying contact information
