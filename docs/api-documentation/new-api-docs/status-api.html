GET /status

Get all active statuses from contacts

Returns grouped statuses by user, excluding expired ones (24-hour limit).

GET /status/user/{userId}

Get all active statuses from a specific user

Returns chronological list of user's active statuses with view tracking.

POST /status/{statusId}/view

Mark a status as viewed

Response

{
  "success": true,
  "data": {
    "status_id": 1,
    "viewed_at": "2025-09-15T00:36:00.000000Z",
    "is_new_view": true
  },
  "message": "Status marked as viewed"
}
GET /status/{statusId}/viewers

Get list of users who viewed a status (status owner only)

Only the status creator can view the list of viewers.

DELETE /status/{statusId}

Delete a status (status owner only)

Deletes the status and associated media files permanently.

💻 Code Examples

JavaScript (Status Management)

// Create text status
const createTextStatus = async (content, bgColor) => {
  const response = await fetch('http://127.0.0.1:8000/api/status', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      type: 'text',
      content,
      background_color: bgColor
    })
  });
  return await response.json();
};

// Create media status
const createMediaStatus = async (file, content) => {
  const formData = new FormData();
  formData.append('type', 'image');
  formData.append('media', file);
  formData.append('content', content);

  const response = await fetch('http://127.0.0.1:8000/api/status', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` },
    body: formData
  });
  return await response.json();
};

// Get all statuses
const getStatuses = async () => {
  const response = await fetch('http://127.0.0.1:8000/api/status', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

// Mark status as viewed
const markAsViewed = async (statusId) => {
  const response = await fetch(`http://127.0.0.1:8000/api/status/${statusId}/view`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}` }
  });
  return await response.json();
};

cURL Commands

# Get all statuses
curl -X GET "http://127.0.0.1:8000/api/status" \
  -H "Authorization: Bearer TOKEN"

# Create text status
curl -X POST "http://127.0.0.1:8000/api/status" \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "text",
    "content": "Hello world!",
    "background_color": "#FF6B6B"
  }'

# Create image status
curl -X POST "http://127.0.0.1:8000/api/status" \
  -H "Authorization: Bearer TOKEN" \
  -F "type=image" \
  -F "media=@/path/to/image.jpg" \
  -F "content=My status"

# Mark status as viewed
curl -X POST "http://127.0.0.1:8000/api/status/1/view" \
  -H "Authorization: Bearer TOKEN"

# Get viewers list
curl -X GET "http://127.0.0.1:8000/api/status/1/viewers" \
  -H "Authorization: Bearer TOKEN"

# Delete status
curl -X DELETE "http://127.0.0.1:8000/api/status/1" \
  -H "Authorization: Bearer TOKEN"

🔧 Integration Notes

  • Statuses automatically expire after 24 hours from creation
  • Use multipart/form-data for media uploads (max 50MB)
  • Implement background color picker for text statuses
  • Generate video thumbnails automatically on upload
  • Track view counts for engagement analytics
