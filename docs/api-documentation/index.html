Authentication

Secure your API requests with Laravel Sanctum Bearer token authentication.

POST /auth/login

Authenticate a user and receive a Bearer token for API access. The token should be included in the Authorization header for all subsequent requests.

Request Body
JSON Request 
{
  "login": "admin@chatapp.com",
  "password": "password"
}
Response
Success Response (200) 
{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": {
      "id": 1,
      "name": "Admin User",
      "email": "admin@chatapp.com",
      "phone_number": "+1234567890",
      "avatar_url": null,
      "is_online": true,
      "last_seen_at": "2025-07-14T19:42:31.000000Z",
      "created_at": "2025-07-14T10:00:00.000000Z"
    },
    "token": "1|abcdef123456789...",
    "token_type": "Bearer"
  }
}

Using Authentication

1. Include Bearer Token

Add the token to the Authorization header for all authenticated requests:

Authorization: Bearer YOUR_TOKEN_HERE
2. Token Expiration

Tokens do not expire by default but can be revoked. Handle 401 responses by redirecting to login.

3. Error Responses

Authentication errors return 401 status with error details:

{
  "success": false,
  "message": "Unauthenticated",
  "error": "Token not provided or invalid"
}
