Status Management

Create, view, and manage status updates with rich styling and privacy controls.

Status Viewing & Analytics

Track who viewed your status and get comprehensive analytics.

POST /status/{statusId}/view Requires Authentication

Mark a status as viewed by the authenticated user. This increments the view count and records the viewer for analytics.

Path Parameters
Parameter Type Required Description
statusId integer Yes The unique identifier of the status
Response
Success Response (200) 
{
  "success": true,
  "message": "Status viewed successfully",
  "data": {
    "status_id": 33,
    "viewer_id": 2,
    "viewed_at": "2025-07-14T19:45:00.000000Z",
    "view_count": 3
  }
}
GET /status/{statusId}/viewers Requires Authentication

Get detailed analytics about who viewed a specific status. Only the status owner can access this information.

Path Parameters
Parameter Type Required Description
statusId integer Yes The unique identifier of the status
Response
Success Response (200) 
{
  "success": true,
  "data": {
    "status_id": "33",
    "viewers": [
      {
        "id": 51,
        "name": "Status Viewer 1",
        "avatar": null,
        "viewed_at": "2025-07-14T19:34:02.000000Z"
      },
      {
        "id": 52,
        "name": "Status Viewer 2",
        "avatar": null,
        "viewed_at": "2025-07-14T19:34:02.000000Z"
      }
    ],
    "total_views": 2
  }
}
DELETE /status/{statusId} Requires Authentication

Delete a status update. Only the status owner can delete their own status. This also removes all associated views and analytics.

Path Parameters
Parameter Type Required Description
statusId integer Yes The unique identifier of the status
Response
Success Response (200) 
{
  "success": true,
  "message": "Status deleted successfully"
}

Status Features

Advanced features and capabilities of the Status API.

24-Hour Auto-Expiration

All status updates automatically expire after 24 hours, just like WhatsApp and Instagram Stories.

  • Automatic cleanup via Laravel scheduler
  • Precise expiration timestamps
  • Time remaining calculations
  • No manual intervention required

Comprehensive Analytics

Track detailed viewer analytics with timestamps and user information.

  • Who viewed your status
  • Exact viewing timestamps
  • Total view counts
  • Viewer profile information

Rich Text Styling

Customize your text status with colors, fonts, and backgrounds.

  • Custom background colors
  • Text color customization
  • Multiple font families
  • Emoji support

Privacy Controls

Control who can see your status updates with flexible privacy settings.

  • Everyone - Public visibility
  • Contacts - Only your contacts
  • Close Friends - Selected users
  • Per-status privacy settings

Usage Examples

1. Creating a Text Status
// Create a colorful text status
const response = await fetch('/api/status', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    type: 'text',
    content: 'ðŸŒŸ Having an amazing day!',
    background_color: '#FF6B6B',
    text_color: '#FFFFFF',
    font_family: 'Arial',
    privacy: 'everyone'
  })
});
2. Viewing a Status
// Mark status as viewed
const viewResponse = await fetch('/api/status/33/view', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
3. Getting Viewer Analytics
// Get who viewed your status
const analytics = await fetch('/api/status/33/viewers', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

const data = await analytics.json();
console.log(`Total views: ${data.data.total_views}`);
console.log('Viewers:', data.data.viewers);
